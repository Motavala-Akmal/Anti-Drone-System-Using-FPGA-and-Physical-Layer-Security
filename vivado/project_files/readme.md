# Vivado Project Files

This folder is used to store selected Vivado project-related files for the FPGA-Based Anti-Drone Security System.

The main purpose of this folder is to keep important Vivado setup files, project notes, and reusable project configuration files without uploading large auto-generated Vivado directories.

---

## Purpose of This Folder

The `project_files/` folder is intended for files that help recreate or understand the Vivado project setup.

This may include:

* Vivado project setup notes
* TCL scripts for project recreation
* Selected constraint files
* Important configuration files
* Project creation instructions
* Board selection information
* Source file mapping notes

---

## Suggested Folder Contents

```text
project_files/
│
├── README.md
├── create_project.tcl
├── source_files.txt
├── board_settings.md
└── constraints_notes.md
```

---

## Recommended Files

### `create_project.tcl`

This file can be used to recreate the Vivado project automatically using a TCL script.

It may include commands for:

* Creating the Vivado project
* Selecting the FPGA board or device
* Adding Verilog RTL files
* Adding testbench files
* Adding constraint files
* Setting the top module
* Running simulation or synthesis

---

### `source_files.txt`

This file can list the source files used in the Vivado project.

Example:

```text
../../verilog/rtl/anti_drone_pls_core.v
../../verilog/rtl/anti_drone_pls_board_demo_top.v
../../verilog/testbench/tb_anti_drone_pls_board_demo_top.v
../../verilog/constraints/board_constraints.xdc
```

---

### `board_settings.md`

This file can describe the FPGA board or device selected in Vivado.

It may include:

```text
Board Name: Basys 3 / Artix-7 / Custom FPGA Board
FPGA Family: Artix-7
Clock Frequency: 100 MHz
Top Module: anti_drone_pls_board_demo_top
Tool Used: Xilinx Vivado
```

---

### `constraints_notes.md`

This file can explain the purpose of the constraint file used for pin mapping and clock definition.

It may include:

* Clock pin mapping
* Reset input mapping
* Mode select switch mapping
* LED output mapping
* Relay trip output mapping
* Charging enable output mapping

---

## Files Not Recommended for Upload

Do not upload large auto-generated Vivado folders because they make the repository heavy and difficult to manage.

Avoid uploading:

```text
.cache/
.hw/
.ip_user_files/
.runs/
.sim/
.gen/
.Xil/
*.jou
*.log
*.str
*.xpr.backup
```

These files are automatically generated by Vivado and are not required for understanding the project.

---

## Recommended GitHub Practice

For a clean GitHub repository, upload only the files that are useful for project understanding and recreation.

Recommended files:

```text
README.md
create_project.tcl
source_files.txt
board_settings.md
constraints_notes.md
```

Avoid uploading complete Vivado project build folders unless they are specifically required.

## Project Recreation Flow

A user should be able to recreate the Vivado project using the source files stored in the repository.

```text
Open Vivado
    |
    v
Create New Project
    |
    v
Select FPGA Board or Device
    |
    v
Add Verilog RTL Files
    |
    v
Add Testbench Files
    |
    v
Add Constraint File
    |
    v
Set Top Module
    |
    v
Run Behavioral Simulation
    |
    v
Run Synthesis
    |
    v
Generate Reports
```

---

## Notes

This folder does not need to contain every Vivado-generated file. It should only contain files that help explain, rebuild, or verify the project.

The main Verilog source files should remain in the root-level `verilog/` folder, while Vivado reports and screenshots should remain inside the `vivado/reports/` and `vivado/screenshots/` folders.

---

## Summary

The `project_files/` folder provides a clean place for Vivado setup resources and project recreation files. It helps keep the repository professional, lightweight, and easy to understand.
